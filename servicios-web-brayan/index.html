1. ¿Qué son los Servicios Web?

Los servicios web son mecanismos de comunicación entre aplicaciones distribuidas que permiten el intercambio de datos mediante protocolos estándar como HTTP. Facilitan la interoperabilidad entre sistemas desarrollados en diferentes lenguajes y plataformas.

Son fundamentales en arquitecturas modernas, microservicios y aplicaciones empresariales.

2. Arquitectura Orientada a Servicios (SOA)

SOA es un modelo arquitectónico que organiza las aplicaciones como un conjunto de servicios independientes, reutilizables e interoperables.

  • Modularidad
  • Escalabilidad
  • Reutilización
  • Interoperabilidad

3. REST vs SOAP

REST

  • Basado en HTTP
  • Formato común: JSON
  • Ligero y flexible
  • Usa URIs para identificar recursos

SOAP

  • Basado en XML
  • Utiliza WSDL
  • Más estructurado y formal
  • Mayor seguridad integrada

4. Protocolos y Estándares

HTTP

Métodos: GET, POST, PUT, DELETE.

Códigos de estado: 200 (OK), 404 (No encontrado), 500 (Error servidor).

JSON

Formato ligero de intercambio de datos utilizado en servicios REST.

XML

Lenguaje de marcado estructurado utilizado principalmente en SOAP.

WSDL

Describe las operaciones disponibles en un servicio SOAP.

UDDI

Directorio para registrar y descubrir servicios web.

5. Implementación REST (Ejemplo práctico)

Un servicio REST utiliza recursos identificados por URIs y opera sobre ellos mediante métodos HTTP.

6. Seguridad en Servicios Web

Autenticación vs Autorización

Autenticación verifica quién eres. Autorización define qué puedes hacer.

JWT

JSON Web Token es un mecanismo de autenticación basado en tokens compuestos por: Header, Payload y Signature.

OAuth2

Protocolo de autorización que permite acceso seguro sin compartir credenciales.

Riesgos comunes

  • CSRF
  • MITM
  • Inyección de código
